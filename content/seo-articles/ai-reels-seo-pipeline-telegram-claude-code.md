Я бы сейчас делал AI-рилзы не как отдельный формат, а как быстрый research loop для SEO.

Не “сгенерил видосик, залил в Reels, помолился алгоритму”, а нормальный пайплайн:

1. найти работающий формат;
2. понять, можно ли повторить его без съемочной команды;
3. сделать короткий видео-эксперимент;
4. измерить saves, shares, comments и переходы;
5. превратить победителя в плотную статью на сайте.

Идея дозрела после канала [@neural_prosecco](https://t.me/neural_prosecco). Уважаемо оставляю ссылку: там много практики про B2C, AI-креаторов, Aesty, трендвотчинг и маркетинг без абстрактной мишуры. Дополнительно полезно читать [@mnk_stories](https://t.me/mnk_stories): там хорошо видно, как founder-storytelling, реальные метрики и маленькие AI-продукты превращаются в контент.

## Суть пайплайна

Главный инсайт: короткое видео здесь не конечная единица контента. Это способ быстро проверить, какую формулировку и какой angle аудитория вообще замечает.

Для `okhlopkov.com` я бы собрал такой цикл:

| Шаг | Что делаем | Что сохраняем |
| --- | --- | --- |
| Трендвотчинг | Смотрим Telegram, X, YouTube, Shorts/Reels и каналы конкурентов | ссылки, hooks, формат, метрики |
| Фильтр | Отбрасываем идеи, которые нельзя повторить за 30-60 минут | complexity score |
| Генерация | Делаем 1-3 коротких видео через AI/скринкаст/CapCut | ролик, промпты, ассеты |
| Постинг | Публикуем в Telegram/X/Shorts/Reels | UTM, дата, канал |
| Вывод | Смотрим saves, shares, comments, CTR, scroll depth | что повторить |
| SEO | Пишем статью, FAQ, source pack, internal links | evergreen URL |

Это похоже на content growth loop, только вместо “контент-плана на месяц” у тебя маленькая лаборатория форматов.

## Что важно смотреть

У `@neural_prosecco` сильный тезис: виральность продукта и размер аккаунта — разные вещи.

Если креатор с 1.5M подписчиков набрал 200k views, это может быть просто охват. Полезнее смотреть аккаунты поменьше, где формат сам вытаскивает результат.

Мой фильтр для AI/tech-контента:

- аккаунт не слишком большой;
- тема рядом с моим доменом: AI agents, Claude Code, Codex, MCP, Telegram automation, GBrain;
- формат можно повторить без команды;
- в комментах есть реальные вопросы, а не только “вау”;
- идею можно потом раскрыть в статье.

Последний пункт самый важный. Если короткое видео не превращается в страницу, source pack или внутреннюю ссылку, оно почти не помогает SEO-машине.

## Какие метрики не врут

Просмотры приятные, но часто бесполезные. Особенно для утилитарных продуктов и сложных тем.

Я бы смотрел так:

| Метрика | Что значит |
| --- | --- |
| Saves / bookmarks | человек хочет вернуться |
| Shares / reposts | формат объясняет боль лучше, чем пользователь сам |
| Comments | есть непонимание, возражения или словарь аудитории |
| Profile clicks / follows | тема цепляет автора, а не только ролик |
| Blog scroll depth | статья реально дочитывается |
| Internal clicks | читатель пошел глубже по кластеру |
| Search impressions | тема начинает жить дольше ленты |

Для Telegram можно использовать прокси: репосты, комменты, реакции, сохранение в обсуждениях и переходы на сайт. Для сайта — GA4/Metrika events: scroll, CTA, outbound, internal nav.

## AI-креаторы — это не замена автору

Посты про Aesty хорошо показывают, зачем вообще нужны AI-креаторы. Не потому что “нейросети заменили людей”, а потому что production стал итеративным.

Human UGC дорогой и медленный: найти человека, объяснить задачу, согласовать правки, дождаться результата. AI-креатор быстрее, но у него другая проблема: надо держать persona, outfit, lore, контент-план и связь с продуктом.

Для моего сайта это значит вот что:

- голос и judgment остаются моими;
- AI может делать визуальный слой, черновые видео, обложки и вариации;
- Codex/Claude Code помогают собрать промпты, source pack, статью и публикацию;
- GBrain хранит, что сработало и почему.

Если AI-креатор начинает звучать как generic рекламный аватар, формат умер. Лучше короткий честный скринкаст с нормальным выводом.

## Мини-боты и сайд-проекты сильнее обычных постов

Самый полезный паттерн из `@mnk_stories` — не “напишите пост”, а “сделайте маленький продукт, который сам становится контентом”.

Пример Aesty Cover Stories: пользователь кидает ссылку на Twitter/Instagram/статью и получает обложку журнала с собой. Это уже не просто ролик. Это shareable artifact.

Для `okhlopkov.com` аналогичные маленькие штуки: AI stack audit card, Claude Code setup checker, Telegram channel wrapped, GBrain context map, prompt-to-agent checklist или калькулятор “сколько токенов ты сжигаешь впустую”.

Дальше схема простая: инструмент → короткое видео → Telegram/X пост → статья “как я это сделал” → internal links в AI-agent кластер.

## Как бы я запустил первый эксперимент

Я бы начал не с Reels, а с темы, где уже есть мой контентный moat: Claude Code / Codex / GBrain.

План на один день: собрать 20-30 референсов из Telegram/X/YouTube, отфильтровать 5 форматов без съемки, поставить каждому score по pain/novelty/repeatability/SEO potential, сделать 2 коротких ролика, запостить и измерить не views, а saves/shares/comments.

Первые два ролика я бы сделал такими: “AI-агент сам задеплоил не то, пока я отошел за чаем” и “как я превращаю Telegram-пост в SEO-статью через Codex + GBrain”.

Эта статья как раз первый тест последнего пункта.
