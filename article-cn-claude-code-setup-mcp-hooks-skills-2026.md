Claude Code是Anthropic的AI代理，在终端里运行。能读文件、执行命令、搜索网页、调用API。你描述你要什么，它搞定怎么做。我用了4个月，这是我的真实配置——不是文档翻译，是我实际怎么用的。 Claude Code到底是什么（不是什么） Claude Code不是聊天机器人，不是自动补全。基本上就是终端里跑的 while True 循环——接任务，选工具，跑一下，看看结果怎样，想下一步该干嘛。一直循环到搞定或者卡住来问你。 工具都是真的：文件读写、 bash 、网页搜索、MCP服务器。中间没有IDE。进项目目录，打 claude ，就这样——你在跟一个能看到整个代码库并且真的能操作的东西对话。 价格：$20/月Pro计划有限额。$100/月Max无限制——我用这个。Claude Code本身没有免费版。需要Claude订阅，然后用 npm install -g @anthropic-ai/claude-code 安装。 我每天怎么用Claude Code 我的工作方式跟传统写代码完全不同。我在Telegram录一条语音，描述我需要什么。我的服务器代理（Claude Code跑在Hetzner VPS上，24/7运行）接收语音转文字，开始干活。 做数据分析，我描述问题然后代理自己去干。写Dune的SQL、拉数据、画图表、组报告。以前手写SQL加格式化要3-4小时，现在说20分钟的话加审查就行。 写代码也一样。说我要什么，代理读文件、写代码、跑测试、commit。我看diff——通常直接接受，偶尔用文字让它改改。 以前我得开15个浏览器标签：文档、Stack Overflow、GitHub issues、API参考。现在一个终端窗口。代理替我读文档。 我的CLAUDE.md——让一切运转的文件 每个Claude Code项目根目录有个 CLAUDE.md 文件。代理启动时先读它。把它想成代理的操作系统，不是prompt。 我的一部分（总共106行）： # Personal Obsidian Vault ## Architecture - **Keep files short** (&lt;150 lines). Use [[links]] for navigation. - **AI never edits raw content.** Voice notes stay pristine. ## 语音消息 收到语音时： - 如果关于具体项目 → 存到: projects/X/ai-docs/ - 如果是私人的 → 存到: personal/diary/ 然后： 1. 在原文下面加标签和项目链接 2. 提取TODO → tasks.md 3. 事实 → overview.md 这个文件路由一切。语音、任务、凭证、项目结构——代理按这些规则搞清楚东西往哪放。差不多每周更新一次，工作流变了就改。 关键理解： CLAUDE.md 不是告诉代理它 是 什么，而是告诉它你的世界怎么 运转 。项目结构、命名规范、文件位置、禁区。更详细的内容在 Building a Second Brain with Obsidian and Claude Code 。 我每天用的MCP服务器 MCP（Model Context Protocol）是让Claude Code连接外部服务的方式。在 .mcp.json 里配好服务器，代理就多了标准文件和bash之外的新能力。 我的 .mcp.json ： { "mcpServers": { "coolify": { "command": "npx", "args": ["-y", "@masonator/coolify-mcp"], "env": { "COOLIFY_ACCESS_TOKEN": "${COOLIFY_ACCESS_TOKEN}", "COOLIFY_BASE_URL": "${COOLIFY_BASE_URL}" } }, "telegram": { "command": "python", "args": ["telegram_mcp_proxy.py"], "env": {} } } } Coolify MCP ——我的部署服务器。代理能部署应用、重启服务、查看日志、管理数据库。更新代理配置时，它自己部署自己。一个能自己部署到生产环境的AI代理。 Telegram MCP ——完整的Telegram访问。读消息、发回复、管理频道。我的服务器代理用它在我睡觉时回答频道讨论里的问题。 还用 Codex MCP 做双重审查。Claude写计划，发给OpenAI的Codex做独立审查，然后整合反馈。两个AI代理互相检查。写过一篇详细的： Claude Code + Codex Dual Review 。 Skills——给代理超能力的Markdown文件 Skill是 ~/.claude/skills/ 里的 .md 文件。没有SDK，没有API，不需要编译。用markdown写指令，Claude Code在相关时候就会遵循。 我做的skill： ton-analyst ——给TON区块链分析写Dune SQL查询。知道schema，知道坑（比如必须过滤 direction = 'in' 避免重复计数），会轮询Dune API拿结果。 ton-profiler ——调查TON钱包关联。调用内部API，构建图数据，识别相关地址集群。 crosspost ——在Telegram、Ghost博客、Twitter/X上用三种语言（俄/英/中）发布内容。 ton-analyst的skill文件大约200行markdown。SQL模板、API端点、常见错误、示例查询。没有代码——只有代理解释并执行的指令。 我在创建skill时犯了很多错。如果你刚开始，先看 写Claude Code Skills的7个常见错误 。 Hooks——每个操作都会触发的代码 Hook是在Claude Code特定事件时自动触发的shell命令。commit前、工具调用后、代理碰某些文件时——基本上什么都能拦截。 我的例子——阻止代理提交敏感文件的pre-commit hook： # .claude/hooks/pre-commit.sh if git diff --cached --name-only | grep -qE '\.(env|key|pem)$|creds\.md'; then echo "BLOCKED: Attempting to commit sensitive files" exit 1 fi 正是hook让Claude Code能无人值守地运行。我的服务器代理24/7跑着，我不担心它泄露凭证或push到错误分支——因为hook不让。 子代理和团队 Claude Code能启动子代理——相当于一个会委派任务的leader。主代理把活儿拆开，分给几个worker，它们各自在自己的上下文窗口里并行干。 我用这个做区块链研究。主代理收到"分析这个钱包最近30天的活动"。它启动： 数据代理——在Dune上写和执行SQL 画像代理——查我们内部的钱包数据库 报告代理——拿两个输出写最终分析 三个代理并行，各有自己的上下文。主代理协调。本来要半天的事，15分钟搞定。 Team更进一步——多个命名代理共享任务列表，在同一个项目上协作。我用team做过大型研究报告：一个代理收集数据，一个写章节，一个做质量审查。 Claude Code vs Cursor——什么时候用哪个 我还装着Cursor。什么时候用什么： Cursor赢在： 快速行内编辑——看到代码，选一块，要求修改。可视化，快速。 探索不熟悉的代码库——Cursor的UI让你边点文件边听AI解释。 确切知道改什么在哪——指比描述快。 Claude Code赢在： 多工具操作——任务涉及读文件+bash+API+改代码。Cursor做不到全部。 自主工作流——后台代理、服务器部署、定时任务。Cursor需要你坐在面前。 大规模修改——重构20个文件，每次改后跑测试。终端比GUI快。 外部系统集成——MCP让代理跟Telegram对话、部署到生产、查询数据库。 我的比例：90% Claude Code，10% Cursor。但我主要做数据分析和代理编排，不做UI。如果整天写React组件，Cursor可能会赢。 费用——$100/月值不值？ Claude Pro（$20/月）给你有限的Claude Code访问。适合尝试、学习工作流、跑简单任务。忙的时候会撞到限额。 Claude Max（$100/月）取消限制。我用Pro两周后就转了，因为每天都撞到上限。$100，我从来不想用量——直接用就是了。 值不值？我工作一小时远超$50。工具每月省2小时就回本了。实际上每 天 省2小时以上。光SQL查询以前就要好几小时。数据格式化、图表、写报告——现在代理全包了。 2026年2月的真实数据：我让代理做区块链研究，产出30页报告含15个图表和40+条SQL查询。手动做要整整一周。代理一个晚上搞定，我审查输出。 「第二大脑」模式 以上所有通过一个想法串联：代理活在我的Obsidian vault里。笔记、项目文件、任务、语音转录——都在一个目录。Claude Code读它们， 知道 我在做什么、什么待办、项目怎么组织。 叫它"第二大脑"因为字面上就是。我把想法倒进语音，代理整理。凌晨2点有灵感，代理早上接手。项目上下文永远不会丢——它在markdown文件里，代理和我都能读。 更深层的版本——把整个职业身份打包成AI能用的格式——我在 dania.zip: Packaging Yourself for AI Agents 里写过。 到底改变了什么 4个月后，最大的变化不是效率。是我不再切换上下文了。以前在SQL编辑器、终端、浏览器、Slack、Notion和代码编辑器之间来回跳。现在坐在一个终端前描述需求，代理搞定剩下的。 不是写更少的代码——大多数任务我 压根不 写代码了。代理写，我审查。工作从"敲代码的人"变成"知道该造什么、能判断对不对的人"。 完整工作流详解看 我的工作流文章 。Obsidian集成看 第二大脑文章 。Skill开发看 这里 。 Daniil Okhlopkov — TON Foundation数据分析负责人。搭建AI代理工作流、区块链分析工具，并记录这一切。 Telegram · Twitter/X · YouTube · Instagram · 小红书
