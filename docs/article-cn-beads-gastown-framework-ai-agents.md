# Beads vs Gastown：是否应该使用外部框架管理AI代理？

> 分析Beads（代理任务追踪器）和Gastown（代理编排）。为什么自建AI代理比使用外部框架更好。

Source: https://okhlopkov.com/cn-beads-gastown-framework-ai-agents/

Beads：极简代理任务追踪器 Beads看起来原生且极简：在.jsonl中追踪任务，结构化，分解，帮助把事情做完。一切都在你的仓库中，上下文不会丢失。 Gastown：编排过度 Gastown则相反：大量文件和角色。我决定不需要它。 为什么不应该使用外部框架 我对代理框架持怀疑态度：你需要研究它们，阅读它们的提示词。但与其安装一堆别人的文件和MCP，我更喜欢挑选真正需要的。 官方第一方才是最好的 Anthropic宣布 受beads启发，将todo升级为完整的任务追踪器。 https://x.com/trq212/status/2014480496013803643 结论： 学习别人的工具，但自己构建你的代理。你不需要别人的遗留代码。
